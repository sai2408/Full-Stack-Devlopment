PRIMARY AND FORIEGN KEY CONSTRAINTS : -
---------------------------------------
THE MAIN REASON TO USE FOREIGN KEY IS TO MAKE RELATION BETWEEN TWO 
OR MORE TABLES

CONDITIONS :-
	1.ONE TABLE SHOULD CONTAIN PRIMARY KEY AND ANOTHER TABLE CONTAINS 
	FOREIGN KEY
	2.A COMMON COLUMN IN BOTH TABLES
	3.THE DATA TYPE OF THOSE COLUMNS MUST BE SAME IN BOTH TABLES

EXAMPLE : -
	--STEP 1
		CREATE TABLE DEPT(DEPTNO INT PRIMARY KEY,DNAME VARCHAR(20),LOCATION VARCHAR(20))
		(PARENT)
	--STEP 2
		INSERT DEPT VALUES (10,'DBA','HYD'),(20,'SAP','MUMBAI'),(30,'HR','PUNE')

		SELECT * FROM DEPT

	--STEP 3
		CREATE TABLE EMPLY(EMPID INT,ENAME VARCHAR(20),SALARY MONEY,DEPTNO INT FOREIGN KEY REFERENCES DEPT(DEPTNO))
		(CHILD)
	--TEST
		INSERT EMPLY VALUES(101,'ADAMS',5500,10) --- ACCEPTED
		INSERT EMPLY VALUES(102,'SAI',3400,40)--- NOT ACCEPTED

PRIMARY KEY :
---------------------
	USED TO IDENTIFY ONE AND ONLY INSTANCE OF ENTITY

FOREIGN KEY:
---------------------
	PRIMARY KEY USED AS A REFERENCE OF ANOTHER TABLE 

CANDIDATE KEY:
------------------------
	AN ENTITY WHICH HAS A CHANCE TO FORM AS A PRIMARY KEY 

SUPER KEY:
----------------
	KEY WHICH CAN UNIQUELY IDENTIFY TUPLE IN A REPLATION
	MINIMAL SUPER KEY IS CALLED AS PRIMARY KEY
